# Визуализатор графа зависимостей Cargo

## 1. Общее описание

CLI-инструмент, который анализирует зависимости пакетов Rust. На втором этапе реализована основная функциональность: программа скачивает удаленный Git-репозиторий (или использует локальный), находит в нем указанный пакет и выводит на экран список его прямых зависимостей.

---

## 2. Описание функций и настроек

Программа настраивается через следующие опции командной строки:

-   `--package <ИМЯ>`: **(Обязательный)** Имя анализируемого пакета.
-   `--repo-url <URL>`: **(Обязательный)** URL-адрес репозитория пакета.
-   `--test-mode`: Флаг, включающий режим работы с тестовым репозиторием.
-   `--ascii-tree`: Флаг, который на будущих этапах будет включать вывод графа в виде ASCII-дерева.
-   `--filter <ПОДСТРОКА>`: Задает подстроку для фильтрации пакетов.
-   `-h`, `--help`: Показывает справочное сообщение со всеми доступными параметрами.

---

## 3. Команды для сборки и запуска

Проект написан на Python и не требует предварительной сборки. Для запуска используется интерпретатор Python.

### Основная команда для запуска:
```
python main.py [ПАРАМЕТРЫ]
```

---

## 4. Примеры использования

### Основной сценарий (анализ удаленного репозитория)

Анализ пакета `serde` из его официального репозитория на GitHub.

Команда:
```
python main.py --package serde --repo-url https://github.com/serde-rs/serde
```

Результат:
```
serde_core
serde_derive
```

### Тестовый режим (анализ локального каталога)

Анализ локального проекта. Необходимо указать флаг `--test-mode` и путь к проекту.

Команда:
```
python main.py --package my-app --repo-url /path/to/your/rust_project --test-mode
```

### Обработка ошибок

Пример вывода при поиске несуществующего пакета в репозитории.

Команда:
```
python main.py --package non-existent-package --repo-url https://github.com/serde-rs/serde
```

Результат:
```
Cargo.toml с пакетом 'non-existent-package' не найден в репозитории.
```
